### Enc-JPG 풀이 ###
-> JPG 파일 만드는 중에 변조됨
-> 문제 파일이 하나 잘못 생성됨
-> 마지막 사진 플래그 글자수는 4개
-> DH{JPG_TXT_PNG}

-> Enc가 flag.jpg의 암호화 하는 파일 아닐까?

jpg의 시그니처
-헤더: FF D8 FF E0
-푸터: FF D9
-> jpg에서 푸터가 2개여서 하나를 지우니까 Flag 일부 발견 가능.

png의 시그니처
-헤더: 89 50 4e 47 0d 0a 1a 0a
-푸터: 49 45 4e 44 ae 42 60 82
-> PNG 파일구조는 파일 시그니처와 다양한 청크들의 집합으로 이루어짐.

*청크: 공통된 데이터를 가지고 있는 데이터들의 집합.
(IHDR, IDAT, PLTE, IEND)
-> 이 중, IHDR, IDAT, IEND는 반드시 있어야함
-> {Length, Chunk Type, Chunk Data, CRC}로 이루어짐.
-> Chunk Data에는 실제 데이터가 들어감
-> IHDR에는 PNG 파일의 기본 정보를 담고 있음. (언제나 13바이트, 시그니처 바로 뒤 위치)
-> 가로, 세로, 비트 깊이, 색 타입, 압축 메소드, 필터링 메소드, 인터레이스 메소드 있음.

-> IDAT에는 실제로 이미지 데이터가 들어감.
-> 여러개의 IDAT 청크 가질 수 있음.

-> DH{How_ENc_ECrypt_yo}

*깨달은 점
1. 푸터가 여러 개면 일부를 없애서 하나로 만들자
2. PNG는 청크로 이루어지고 이를 데이터로 가지기에 PNG의 파일구조를 잘 파악하자.
