### Event Log ###
-> 사용자 로그인, 시스템 시작/종료 등 Windows 운영체제에서 발생하는 이벤트들을 기록한 파일
-> 사용자가 언제 로그인해서, 어떤 프로세스 실행했고, 어떤 서비스를 이용했는지 사건의 재구성 가능
-> 추가로 시스템에서 발생한 오류나 보안 결함에 대해 추적도 좋음

Ex) 악성코드의 동작 과정 파악
-> 계정 생성, 해당 계정에 높은 권한 부여, 특정 커맨드로 프로그램 실행 
-> 이러한 과정을 이벤트 로그에서 파악 가능

- 사용자 계정 생성, 암호 변경, 권한 변경
- 시스템 로그인 성공, 실패
- 프로세스 시작 및 종료
- 시스템 원격 로그인
- 저장장치 연결

## 이벤트 뷰어 ##
-> 이벤트 로그 분석 도구

- Application: 응용 프로그램에서 발생한 이벤트 기록, (작업 성공여부, 오류 여부 기록)
- Security: 파일 열기, 만들기와 같은 시스템 리소스 사용 이벤트, 로그인 성공/실패, 시스템 보안 정책 변경 이벤트, 유일하게 사용자가 기록할 이벤트 유형 변경 가능
- Setup: 응용프로그램 설치나 설정과 관련된 이벤트 기록
- 응용프로그램 및 서비스 로그: Internet Explorer, Microsoft Office, 방화벽, USB 연결 흔적, Wifi 연결 (WLAN) 등 (Windows 제공 기능)

## 이벤트 로그 경로 ##
"C:\Windows\System32\winevt\Logs"

-Security.evtx: 보안
-System.evtx: 시스템
-Application.evtx: 응용 프로그램

-EventID: 이벤트 유형 식별
-Level: 이벤트 심각도
-Security UserID: 이벤트가 발생한 사용자 계정
-TimeCreated SystemTime: 이벤트가 발생한 시스템 시간
-Execution ProcessID/ThreadID: 이벤트를 발생시킨 프로세스/스레드 ID

## 이벤트 로그 수집 ##
-> FTK Imager에서 Logs 폴더 전체 export
-> 이후, 이벤트 뷰어에서 '동작 -> 저장된 로그 열기'

## 이벤트 로그 필터링 ##
-> '이벤트 뷰어 -> 사용자 지정 보기 만들기'
- 12: 시작
- 13: 종료
- 4608: Windows 시작 (지피티는 로그온이라는데)
- 1074: Windows 종료
- 42: 시스템 절전 모드
- 6008: 시스템 비정상 종료
- 4624: 계정 로그온 성공
- 4625: 계정 로그온 실패
- 4634: 계정 로그오프

- 4672: 관리자 권한으로 특수 로그온

- 1102,104: 이벤트 로그 삭제
- 10000, 10100: 장치 드라이버 설치, 업데이트
- 225: 장치 연결 해제
- 1006: 파티션 연결 해제
- 10000, 10001: 네트워크 연결, 해제

"같은 ID여도 내용은 때로는 다를 수 있음. 이에 따라 상세 내용을 확실하게 읽어본 후 사실 관계 파악 필요"
