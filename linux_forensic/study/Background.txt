### Linux Forensics - Background ###

## 들어가며 ##
-> Linux 시스템에 대한 포렌식 분석 방법
-> 활성 시스템 포렌식 개념 (Linux 포렌식에서는 기존과 달리 활성 시스템에서 증거를 수집하고 분석하는 경우가 많음)
-> 활성 시스템에서 증거를 수집해야 하는 경우는 언제인지, 활성화된 시스템에서 증거를 수집할 때 유의해야할 점은 무엇인지 배움.

## 활성 시스템 포렌식 ##

# 활성 시스템 #
-> 전원이 켜져 있는 시스템 (전원이 켜진 상태, 화면 보호기 실행, 절전 모드)
-> Linux 운영체제는 서버용 PC로 많이 사용됨. (24시간 내내 켜져 있는 경우가 많음)

# 활성 시스템 포렌식 #
-> 활성 시스템에서 증거를 수집해서 분석하는 것 (비활성 시스템 포렌식이란 수집된 디스크 이미지에서 증거를 수집해 분석하는 것)
-> 증거를 수집하기 위해 목적에 맞는 파일인지 확인하기 위해 파일이나 폴더를 열어보기도 하고, 
   시스템 내에서 프로그램을 실행 시켜 원하는 정보를 수집하거나 데이터를 가져오기도함
-> 이런 방법은 시스템이 변경될 수 있어서 증거 수집 시 선호되는 방법은 아님.

*활성 시스템 포렌식이 왜 필요한가?

1. 활성 시스템 포렌식을 통해서만 수집할 수 있는 데이터가 존재함. (사용자가 분석 직전 실행 중이던 상태 그대로 확인 가능. 현재 실행 중인 프로그램 및 작업 중이던 파일 모두 파악 가능)
+ 메모리 덤프 수집 가능 (사용자의 암호화 키나 패스워드 포함 가능)
+ 클라우드 서버나 원격 DB에 데이터 저장하는 경우 있기에 원격 시스템의 정보를 반드시 켜저 있는 상태에서 접속해서 데이터를 가져와야함

2. 특정 상황에서는 활성 시스템으로 데이터를 수집할 수 밖에 없음 (웹 서버의 경우가 대표적인 예시)

# 활성 시스템 포렌식 주의점 #
1. CLI 도구가 권장됨.
-> 메모리 사용이 적고 검증된 CLI 도구를 통해 사용

2. 원하는 데이터 검색할 수 있는 방법 미리 마련
-> Windows에서는 everything과 같은 유틸리티, Linux에서는 find, grep 등과 같은 스크립트를 미리 만들어 놓음

3. 사용할 도구는 전부 준비해 가져감
-> 시간 제한 및 시스템 내 프로그램의 악성 코드 가능성

4. 수집 명령어는 운영체제에서 기본적으로 제공하는 것 사용
-> Windows에서는 배치 스크립트 (.bat), Linux에서는 쉘 스크립트 (.sh)

# 리눅스 기본 명령어 #
1. 리다이렉션 > (기존 내용 삭제), >> (기존 파일 내용에 추가)
2. netsta -antp (시스템의 네트워크 연결 상태)
3. ps -ef (시스템에서 실행 중인 프로세스)
4. 기본적으로 .이 붙으면 숨김 파일 처리됨
